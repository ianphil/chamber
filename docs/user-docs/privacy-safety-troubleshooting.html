Privacy, safety, and troubleshooting

Chamber gives minds useful abilities, but you stay in control. Use this guide to understand what stays local, what to avoid sharing, and how to fix common issues.

Safety summary

Keep secrets out of chat, review tool approvals carefully, and only install minds or marketplace sources you trust. If something feels wrong, stop the work first.

What stays on your device

Chamber stores mind folders on your computer so each mind can keep its profile, notes, and local working files. These folders are meant for your use on this device. Do not share a mind folder with people you do not trust.

Credentials

Sign-in credentials are stored in your operating system keychain, not inside mind folders. On Windows this means the Windows credential store. On macOS it means Keychain. On Linux it means your desktop keyring when available.

Mind folders

  • Mind folders can contain conversation context, notes, and files created for that mind.
  • They should not contain passwords, access tokens, private keys, or recovery codes.
  • Back up or delete mind folders with the same care you use for work documents.
Do not paste secrets

Do not paste passwords, access tokens, private keys, payment details, personal identification numbers, or private customer data into chat unless you are certain you are allowed to share it and understand where it may be sent.

Use marketplace sources you trust

Marketplace items can add minds, prompts, views, and helpful setup. Treat them like software from any other source: install only from people or organizations you trust, and remove anything that looks unexpected.

Before installing

  • Check who published the item.
  • Read what the item says it will add or change.
  • Avoid sources sent by strangers or copied from untrusted websites.
  • If your workplace manages approved sources, use those first.

Review tool approvals

Minds may ask to use tools to read files, create files, update views, or do other work. Chamber shows approval prompts so you can decide what is allowed.

How to decide

  • Approve only if the action matches what you asked the mind to do.
  • Read file names, website names, and action descriptions before approving.
  • Deny the request if it mentions a private folder, unknown website, or action you did not expect.
  • Stop the mind if it keeps asking for actions that do not make sense.

Lens and Canvas safety expectations

Lens and Canvas help minds show dashboards, forms, reports, and other visual work. They are designed to be separated from the app's private controls, but you should still use the same caution you use with any generated content.

Lens views

  • Lens views should match the mind or task that created them.
  • If a view looks broken, blank, or unrelated, do not enter sensitive information.
  • Refresh the view or ask the mind to rebuild it if it appears incomplete.

Canvas pages

  • Canvas pages are for reports and dashboards, not password entry.
  • Do not sign in through a Canvas page unless you fully trust the source and expected it.
  • Close unexpected Canvas pages and ask the mind why it opened one.

Troubleshooting common problems

Try the simplest fix first. If a problem started after switching accounts or installing an update, quit and reopen Chamber before changing anything else.

Login problems

  • Make sure your browser finished the GitHub sign-in step.
  • Check that you signed in with the GitHub account you meant to use.
  • If the sign-in window is stuck, close it and start sign-in again from Settings.
  • If your organization requires extra approval, complete that step in GitHub first.

Marketplace access errors

  • Confirm you are online.
  • Switch to the GitHub account that has access to the source.
  • Check whether the source was removed, renamed, or made private.
  • Use only trusted sources if Chamber warns that access is blocked or unavailable.

A mind does not respond

  • Check your internet connection.
  • Send a short test message, such as "hello".
  • Start a new conversation if the current one seems stuck.
  • Quit and reopen Chamber if no minds respond.

An agent times out

  • Ask for a smaller task or split the request into steps.
  • Stop the current work before sending a new request.
  • Try again later if the service may be busy.

Models are missing

  • Check that the active GitHub account has the expected Copilot access.
  • Switch accounts if the wrong account is active.
  • Some models may not be available for every account, plan, or organization.
  • Sign out and sign in again if the model list looks outdated.

Views are broken or blank

  • Refresh the Lens or Canvas view.
  • Ask the mind to rebuild the view in simpler form.
  • Do not enter sensitive information into a broken or unexpected view.
  • Restart Chamber if all views stop loading.

Update or install issues

  • Quit Chamber fully, then open it again and retry the update.
  • Make sure you have permission to install apps on this computer.
  • Check that security software is not blocking the installer.
  • Download only from the official project page or a trusted workplace source.

If you are unsure what the updater icon means, see the Auto Updater guide.

When in doubt

Stop the mind, do not approve new tool requests, and save any important work. Then reopen Chamber and try a smaller request.
